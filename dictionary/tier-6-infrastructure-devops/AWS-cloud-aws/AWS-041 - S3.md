# S3

## ⚡ TL;DR

**S3 (Simple Storage Service)** is AWS's infinitely scalable object storage. Store any file (object) in named buckets. 99.999999999% (11 9s) durability. Replicated across ≥3 AZs automatically. Key features: versioning, lifecycle policies, event notifications, cross-region replication, static website hosting, and fine-grained access via bucket policies + IAM. Foundation for data lakes, backups, artifacts, and static assets.

---

## 🔥 Problem This Solves

You need to store files: backups, user uploads, build artifacts, logs, ML training data, static website files. Without S3: manage NFS servers, worry about disk space, handle replication, implement versioning. S3: unlimited storage, no capacity planning, 11 9s durability, automatic replication, pay per GB.

---

## 📘 Textbook Definition

Amazon S3 is an object storage service offering industry-leading scalability, data availability, security, and performance. Objects are stored in buckets, with each object having a key (path), value (data), metadata, and version ID (if versioning enabled). S3 provides 99.99% availability and 99.999999999% durability.

## ⏱️ 30 Seconds

```
Bucket:  globally unique name container (per region)
Object:  key + value + metadata + version
  Max object size: 5TB (multipart upload required >5GB)

Access control:
  Bucket Policy: JSON policy attached to bucket (resource-based)
  IAM Policy:    attached to user/role (identity-based)
  ACLs:          legacy, largely replaced by policies
  Block Public Access: account/bucket-level kill switch

S3 URI:         s3://bucket-name/prefix/object-key
HTTPS URL:      https://bucket.s3.amazonaws.com/key
                https://bucket.s3.region.amazonaws.com/key
```

## 🔩 First Principles

- **Object storage**: not a file system; no folders (just key prefixes that look like folders)
- **Eventual consistency**: S3 provides **strong consistency** for new writes since 2020 (read-after-write)
- **Flat namespace**: "folders" are just key prefixes; `uploads/2024/file.jpg` is one key
- **Replication**: data stored in ≥3 AZs in the region; durability ≠ version protection
- **Public access**: Block Public Access is default ON; intentional to prevent data leaks

## 🧪 Thought Experiment

User uploads a profile photo. Frontend calls your API → API stores photo in S3 (`profiles/{user_id}/avatar.jpg`). API returns a pre-signed URL with 1hr expiry → browser loads photo directly from S3. You never handled file bytes in your app server. CloudFront CDN in front → cached at edge → 10ms globally. Versioning on → users can revert to previous avatar.

## 🧠 Mental Model / Analogy

S3 is an **infinitely large filing cabinet**: each drawer (bucket) contains labeled files (objects). Files can be any size, any content. You access files by their label (key). The cabinet is fireproof (11 9s durability), has a security lock (IAM/bucket policy), and can automatically move old files to cheaper storage (lifecycle policies).

## 📶 Gradual Depth

**Level 1 - Beginner**: Create bucket, upload objects via console or CLI. Set bucket policy for access control. Enable versioning to protect against accidental deletion.

**Level 2 - Practitioner**: Pre-signed URLs: generate time-limited URLs for object access without exposing credentials. S3 Event Notifications → Lambda (trigger processing on upload). Static website hosting. Cross-Region Replication for DR.

**Level 3 - Advanced**: S3 Inventory: generate daily/weekly CSV of all objects + metadata (without listing API calls). S3 Storage Lens: cross-bucket analytics. Multipart upload for large files. S3 Select: query CSV/JSON data with SQL without downloading full object. S3 Batch Operations: execute operations (copy, tag, invoke Lambda) on billions of objects.

**Level 4 - Expert**: S3 Access Points: create named endpoints with custom access policies (per-team access to same bucket). S3 Object Lambda: transform objects on GET (resize images, redact PII on the fly). Requester Pays: object requester pays data transfer cost (useful for public datasets). S3 Intelligent-Tiering: automatic tier movement based on access patterns without retrieval fee. S3 Transfer Acceleration: uploads to nearest CloudFront edge → routed to S3 via AWS backbone (30-300% faster uploads).

## ⚙️ How It Works

### S3 Operations (Java AWS SDK v2)

```java
// Spring Boot: S3 operations with AWS SDK v2
@Service
public class FileStorageService {

    private final S3Client s3Client;
    private final S3Presigner presigner;
    private final String bucketName;

    public FileStorageService(S3Client s3Client, S3Presigner presigner,
                              @Value("${aws.s3.bucket}") String bucketName) {
        this.s3Client = s3Client;
        this.presigner = presigner;
        this.bucketName = bucketName;
    }

    // Upload file
    public String upload(String key, byte[] data, String contentType) {
        s3Client.putObject(
            PutObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .contentType(contentType)
                .serverSideEncryption(ServerSideEncryption.AES256)  // encryption
                .build(),
            RequestBody.fromBytes(data)
        );
        return key;
    }

    // Download file
    public byte[] download(String key) {
        ResponseBytes<GetObjectResponse> response = s3Client.getObjectAsBytes(
            GetObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build()
        );
        return response.asByteArray();
    }

    // Generate pre-signed URL (for browser download, 1 hour)
    public URL generatePresignedGetUrl(String key) {
        PresignedGetObjectRequest presignedRequest = presigner.presignGetObject(
            GetObjectPresignRequest.builder()
                .signatureDuration(Duration.ofHours(1))
                .getObjectRequest(r -> r.bucket(bucketName).key(key))
                .build()
        );
        return presignedRequest.url();
    }

    // Generate pre-signed URL for upload (browser direct upload)
    public URL generatePresignedPutUrl(String key, String contentType) {
        PresignedPutObjectRequest presignedRequest = presigner.presignPutObject(
            PutObjectPresignRequest.builder()
                .signatureDuration(Duration.ofMinutes(15))
                .putObjectRequest(r -> r
                    .bucket(bucketName)
                    .key(key)
                    .contentType(contentType))
                .build()
        );
        return presignedRequest.url();
    }

    // Delete object
    public void delete(String key) {
        s3Client.deleteObject(
            DeleteObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build()
        );
    }
}
```

### Bucket Policy (Terraform)

```hcl
# Block all public access (security baseline)
resource "aws_s3_bucket_public_access_block" "app" {
  bucket                  = aws_s3_bucket.app.id
  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

# Bucket policy: allow specific role + enforce encryption
resource "aws_s3_bucket_policy" "app" {
  bucket = aws_s3_bucket.app.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Sid    = "AllowAppRole"
        Effect = "Allow"
        Principal = {
          AWS = aws_iam_role.app.arn
        }
        Action   = ["s3:GetObject", "s3:PutObject", "s3:DeleteObject"]
        Resource = "${aws_s3_bucket.app.arn}/*"
      },
      {
        Sid    = "DenyUnencryptedUpload"
        Effect = "Deny"
        Principal = "*"
        Action   = "s3:PutObject"
        Resource = "${aws_s3_bucket.app.arn}/*"
        Condition = {
          StringNotEquals = {
            "s3:x-amz-server-side-encryption" = "AES256"
          }
        }
      },
      {
        Sid    = "DenyHTTP"
        Effect = "Deny"
        Principal = "*"
        Action   = "s3:*"
        Resource = [
          aws_s3_bucket.app.arn,
          "${aws_s3_bucket.app.arn}/*"
        ]
        Condition = {
          Bool = { "aws:SecureTransport" = "false" }
        }
      }
    ]
  })
}

# Enable versioning
resource "aws_s3_bucket_versioning" "app" {
  bucket = aws_s3_bucket.app.id
  versioning_configuration {
    status = "Enabled"
  }
}

# Server-side encryption (default)
resource "aws_s3_bucket_server_side_encryption_configuration" "app" {
  bucket = aws_s3_bucket.app.id
  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "aws:kms"
      kms_master_key_id = aws_kms_key.s3.arn
    }
    bucket_key_enabled = true  # reduces KMS API calls/cost
  }
}
```

### S3 Event Notification → Lambda

```hcl
# Trigger Lambda on new object upload
resource "aws_s3_bucket_notification" "upload" {
  bucket = aws_s3_bucket.uploads.id

  lambda_function {
    lambda_function_arn = aws_lambda_function.process_upload.arn
    events              = ["s3:ObjectCreated:*"]
    filter_prefix       = "uploads/"
    filter_suffix       = ".jpg"  # only JPEG uploads
  }
}
```

## ⚠️ Common Misconceptions

| Misconception                       | Reality                                                                                      |
| ----------------------------------- | -------------------------------------------------------------------------------------------- |
| "S3 folders are real directories"   | Folders are key prefixes; no real directory structure                                        |
| "All S3 regions are the same price" | S3 pricing varies by region; us-east-1 is cheapest                                           |
| "Public access is default off"      | Block Public Access is default ON; but old buckets may have been created before this default |
| "S3 is eventual consistent"         | S3 provides strong read-after-write consistency since Dec 2020                               |

## 🧠 Think About This

The most consequential S3 security decision is the bucket access model. Public data leaks from S3 have compromised millions of records across organizations. The defense is: (1) Enable Block Public Access at the **account level** (not just bucket level) - this is the nuclear option that prevents any S3 bucket in the account from being made public, even if someone accidentally sets a public bucket policy. (2) Enable S3 GuardDuty S3 protection to detect anomalous access patterns. (3) Regular AWS Config rule check: `s3-bucket-public-read-prohibited`. The root cause of most S3 leaks is not malicious intent but operational mistake - a developer making a bucket public for testing and forgetting to revert. Account-level Block Public Access closes this entire category of risk with one setting.
